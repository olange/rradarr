Jeux de fichiers DICOM de test

Le dossier test/ contient plusieurs jeux d’images DICOM destinés aux tests fonctionnels (voir script test/exam_spec.rb), répartis dans les sous-dossiers suivants:

invalid-dicom/

contient 1 fichier DICOM corrompu (créé à partir des ~140 premiers octets d’un fichier DICOM valide).

mixed-content/

contient 3 fichiers DICOM, dont l’un est une image scout, les autres des images d’examen, ainsi que deux fichiers texte; destinés à éprouver la capacité de la classe Exam à les identifier correctement.

no-dicom/

dossier vide (aucun fichier DICOM ou autre).

non-homegeneous/

contient 6 fichiers DICOM provenant de 3 séries d’images différentes, dont les deux premiers n’ont pas la même structure de données que les quatre suivants (68 éléments vs 226 éléments).

Les deux premières images proviennent du rapport de dose, deux autres de l’examen 100KV et les deux derniers de l’examen 140KV.

other/

un fichier DICOM minimal, ne contenant que l’en-tête avec la signature.

valid-dicom/

LAPIN2_PRO_APC_100KV_CTDIVOL8_6/

15 images DICOM d’une radiographie en tranches d’un lapin, réalisée avec l’exposition automatique du scanner (sous-ensemble, le jeu complet comprend 1000 images). Contiennent 226 éléments de données.

LAPIN2_PRO_APC_140KV_CTDIVOL2_16/

10 images DICOM d’une radiographie en tranches du même lapin, avec une exposition constante du scanner (sous-ensemble, le jeu complet comprend 1000 images). Contiennent 226 éléments de données.

Localizers_1/

2 images DICOM, avec une radiographie transversale du même lapin, donnant un aperçu en plan de toutes les tranches. Contiennent 218 éléments de données.

Rapport_dose_999/

2 images DICOM avec une indication des doses de radiations émises lors des précédentes radiographies. Ces images ne présentent que des indications littérales (texte), sous forme de bitmap. Contiennent 68 éléments de données.
